Explain the difference between hard handover and soft handover in mobile networks. Give one scenario where each may occur.
  • Hard handover ("break before make"): The old connection is released before the new one is established. Brief signal interruption possible.
  • Soft handover ("make before break"): The new connection is established before releasing the old one. Seamless transition; used in systems that support parallel connections (e.g., CDMA-based UMTS).
  • Hard handover example: Switching to a cell using a completely different frequency (no overlap).
  • Soft handover example: Moving between two UMTS cells with overlapping coverage at the cell edge.

What is the high-level purpose of the GSM authentication / session key / encryption sequence?
  • The three-step GSM security flow:
    1. Authentication: Verify the user/device (using shared secret + challenge-response).
    2. Session key derivation: Compute a temporary session key from the authentication exchange.
    3. Encryption: Use the session key to encrypt subsequent communication over the air.

What is the conceptual difference between the GSM Incoming Call and GSM Outgoing Call signaling flows?
  • Incoming call: The network initiates — it pages the device and sets up the call toward the user.
  • Outgoing call: The user/device initiates — it sends a call setup request toward the network.
  • Both involve: channel request → assignment → authentication → connection setup, but the trigger and direction differ.
